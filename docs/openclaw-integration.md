# OpenClaw Integration

Pluribus has built-in support for [OpenClaw](https://openclaw.ai) — an AI agent runner that uses `AGENTS.md` for project-specific context and instructions.

## How it works

When you run `pluribus sync` with the `openclaw` skill, Pluribus generates an `AGENTS.md` file from your `pluribus.md` context. The output format follows OpenClaw's expected structure for agent configuration.

### Sync command

```bash
# Generate AGENTS.md
pluribus sync --tools openclaw

# Generate all tools including AGENTS.md
pluribus sync
```

### Output file

| Source | Output |
|---|---|
| `pluribus.md` | `AGENTS.md` |

## Sections mapping

Pluribus maps your `pluribus.md` sections to `AGENTS.md` as follows:

| pluribus.md section | AGENTS.md section | Required |
|---|---|---|
| `# Identity` | `## Identity` | ✅ Yes |
| `# Stack` | `## Stack` | ✅ Yes |
| `# Conventions` | `## Conventions` | ✅ Yes |
| `# Goals` | `## Goals` | ✅ Yes |
| `# Constraints` | `## Constraints` | ✅ Yes |
| `# Examples` | `## Examples` | Optional |
| `# Context` | `## Context` | Optional |
| `# Anti-patterns` | Appended to `## Context` | Optional |

## Example

Given a `pluribus.md` with:

```markdown
# Identity
I am Helios, an AI assistant for personal research and development automation.

# Stack
- Runtime: OpenClaw (latest)
- OS: Windows 11, PowerShell
- Primary channel: Telegram

# Conventions
- Be direct, no preamble
- Prefer structured output

# Goals
1. Maximize personal leverage
2. Zero noise

# Constraints
- Never send messages without explicit confirmation
- Never expose API tokens in logs
```

Running `pluribus sync --tools openclaw` generates:

```markdown
<!-- Generated by Pluribus 0.1.0 on 2026-03-09 - do not edit manually -->
<!-- Source: pluribus.md -->

# AGENTS.md

## Identity
I am Helios, an AI assistant for personal research and development automation.

---

## Stack
- Runtime: OpenClaw (latest)
- OS: Windows 11, PowerShell
- Primary channel: Telegram

---

## Conventions
- Be direct, no preamble
- Prefer structured output

---

## Goals
1. Maximize personal leverage
2. Zero noise

---

## Constraints
- Never send messages without explicit confirmation
- Never expose API tokens in logs
```

See [`examples/openclaw/`](../examples/openclaw/) for a full working example.

## Why OpenClaw uses AGENTS.md

OpenClaw reads `AGENTS.md` as the primary source of truth for:
- **Agent identity and persona** — who the agent is, how it behaves
- **Tech stack** — what tools, languages, and services are in use
- **Conventions** — coding standards, naming, structure
- **Goals** — what the agent optimizes for
- **Constraints** — hard rules the agent must not violate

Keeping this in sync with your other context files (CLAUDE.md, .cursorrules) manually is error-prone. Pluribus eliminates this by treating `AGENTS.md` as just another output of your canonical `pluribus.md`.

## Custom skill override

You can customize the AGENTS.md template by creating a local skill file:

```
pluribus/skills/openclaw.md
```

See [`spec/skills-format.md`](../spec/skills-format.md) for the skill file format and available template variables.
